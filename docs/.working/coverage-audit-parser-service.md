# Аудит покрытия parser-service-документацией

## Вердикт
Покрытие не полное. Основная форма runtime задокументирована хорошо, но есть 4 конкретных gap/mismatch относительно текущей реализации `service`.

## Наблюдения
1. Поведение `probe` cancel для not-found задокументировано как `404`, но текущий route этот path не реализует.
Code: [service/app/api/v1/sync.py](/home/andrey-debian/Projects/wardrobe-parser-platform/service/app/api/v1/sync.py:475) сразу разыменовывает `job` после `probe_orchestrator.cancel(job_id)` и не имеет `None` guard, в отличие от [service/app/api/v1/sync.py](/home/andrey-debian/Projects/wardrobe-parser-platform/service/app/api/v1/sync.py:504).
Docs, которые заявляют generic unknown job `404`: [docs/parser-service-design/07-backend-contracts-and-weight-enrichment.md](/home/andrey-debian/Projects/wardrobe-parser-platform/docs/parser-service-design/07-backend-contracts-and-weight-enrichment.md:119).
Impact: задокументированная HTTP-boundary шире, чем подтвержденная реализация для `POST /api/v1/sync/probe/jobs/{job_id}/cancel`.

2. Покрытие `product_batch` не описывает жесткие forwarding filters и часть concrete item shape.
Code строит поля item в [service/app/services/sync_orchestrator_service.py](/home/andrey-debian/Projects/wardrobe-parser-platform/service/app/services/sync_orchestrator_service.py:202) и форвардит только items, у которых сохраняются `source_product_url` и непустой массив `variants[]`, в [service/app/services/sync_orchestrator_service.py](/home/andrey-debian/Projects/wardrobe-parser-platform/service/app/services/sync_orchestrator_service.py:287). Тот же блок также несет `canonical_url`, `external_id`, `unavailable_reason`, `buyer_total_price` и `buyer_service_fee`.
Docs покрывают существование batch и правило `missing_weight`, но не эти forwarding gates и не более полную форму item contract: [docs/parser-service-design/07-backend-contracts-and-weight-enrichment.md](/home/andrey-debian/Projects/wardrobe-parser-platform/docs/parser-service-design/07-backend-contracts-and-weight-enrichment.md:75), [docs/api/data-models/sync-event.md](/home/andrey-debian/Projects/wardrobe-parser-platform/docs/api/data-models/sync-event.md:36), [docs/api/protocols/backend--parser-service.md](/home/andrey-debian/Projects/wardrobe-parser-platform/docs/api/protocols/backend--parser-service.md:124).

3. Документация по payload событий слишком поверхностна для текущего orchestrator contract.
Code публикует structured payload для:
- `source_started` с `source_index`, `total_sources`, `attempt`, `stage_code`, `stage_label`, `progress_percent` в [service/app/services/sync_orchestrator_service.py](/home/andrey-debian/Projects/wardrobe-parser-platform/service/app/services/sync_orchestrator_service.py:368)
- `source_progress` с `strategy`, raw `fields`, `products_success`, `products_error`, `progress_percent` в [service/app/services/sync_orchestrator_service.py](/home/andrey-debian/Projects/wardrobe-parser-platform/service/app/services/sync_orchestrator_service.py:429)
- прогресс requeue через `stage="source_requeued"` плюс `max_attempts` в [service/app/services/sync_orchestrator_service.py](/home/andrey-debian/Projects/wardrobe-parser-platform/service/app/services/sync_orchestrator_service.py:474) и [service/app/services/sync_orchestrator_service.py](/home/andrey-debian/Projects/wardrobe-parser-platform/service/app/services/sync_orchestrator_service.py:548)
- terminal payload job с `reason`, `failed_sources`, `stage_code`, `stage_label` в [service/app/services/sync_orchestrator_service.py](/home/andrey-debian/Projects/wardrobe-parser-platform/service/app/services/sync_orchestrator_service.py:592).
Docs в основном просто перечисляют типы событий, не публикуя эти payload fields и не упоминая variant stage `source_requeued`: [docs/parser-service-design/03-sync-api-and-orchestrator-runtime.md](/home/andrey-debian/Projects/wardrobe-parser-platform/docs/parser-service-design/03-sync-api-and-orchestrator-runtime.md:136), [docs/api/data-models/sync-event.md](/home/andrey-debian/Projects/wardrobe-parser-platform/docs/api/data-models/sync-event.md:24), [docs/api/protocols/backend--parser-service.md](/home/andrey-debian/Projects/wardrobe-parser-platform/docs/api/protocols/backend--parser-service.md:113).

4. Документация по browser-runner не точна относительно текущей runtime/config surface.
Scenario ids в коде: `default-shopify-sitemap` и `dolcevitahub-shopify-sitemap`: [service/browser_runner/src/scenarios/default-shopify.mjs](/home/andrey-debian/Projects/wardrobe-parser-platform/service/browser_runner/src/scenarios/default-shopify.mjs:3), [service/browser_runner/src/scenarios/dolcevitahub.mjs](/home/andrey-debian/Projects/wardrobe-parser-platform/service/browser_runner/src/scenarios/dolcevitahub.mjs:3), [service/browser_runner/src/scenarios/registry.mjs](/home/andrey-debian/Projects/wardrobe-parser-platform/service/browser_runner/src/scenarios/registry.mjs:1). Docs сейчас называют их `default-shopify` и `dolcevitahub`: [docs/parser-service-design/06-browser-runner-and-automation.md](/home/andrey-debian/Projects/wardrobe-parser-platform/docs/parser-service-design/06-browser-runner-and-automation.md:93).
Python browser strategy считает `max_sitemaps` опциональным и также читает `shopify_currency.country_code` для `--country-code`: [service/app/strategies/shopify_browser_extension.py](/home/andrey-debian/Projects/wardrobe-parser-platform/service/app/strategies/shopify_browser_extension.py:44), [service/app/strategies/shopify_browser_extension.py](/home/andrey-debian/Projects/wardrobe-parser-platform/service/app/strategies/shopify_browser_extension.py:52), [service/app/strategies/shopify_browser_extension.py](/home/andrey-debian/Projects/wardrobe-parser-platform/service/app/strategies/shopify_browser_extension.py:167). Docs перечисляют `max_sitemaps` среди required keys и не называют `country_code`: [docs/parser-service-design/06-browser-runner-and-automation.md](/home/andrey-debian/Projects/wardrobe-parser-platform/docs/parser-service-design/06-browser-runner-and-automation.md:35).
Текущая config также хранит `browser_extension.includeLocaleSitemaps` для `dolcevitahub.com`: [service/config/sources.json](/home/andrey-debian/Projects/wardrobe-parser-platform/service/config/sources.json:1245). Этот ключ описан в [docs/parser-service-design/02-source-registry-and-configuration-model.md](/home/andrey-debian/Projects/wardrobe-parser-platform/docs/parser-service-design/02-source-registry-and-configuration-model.md:122), но `ShopifyBrowserExtensionStrategy` его не читает и не прокидывает дальше, поэтому docs сейчас завышают реальное active runtime usage.
